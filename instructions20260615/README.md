## なぜこの 2 本か

- **I1 が高優先**：`RESULTS.md` は E-A3 を「未実装・out of scope」と明記しているのに、
  `paper_draft.md` の §8 は E-A3 を「機械検証した計算的対応物」として書いている。
  この**論文と実装の不一致**は、査読者が companion を覗けば気づく。実装して塞ぐ。
- **I2 は探索**：算術側が stage 1 のみなのは GL 梯子に比べ非対称。ただし反復拡張の
  超限化は Turing–Feferman そのもので非自明。だから「Foundation の範囲で登れるだけ登り、
  尽きたら止まって理由を報告」。**新規 formalization には突入しない**（歯止けは I2 内に明記）。
  stage 1 が天井でも、possible-infinity の主張上は失敗ではない（理由も I2 内に明記）。

## 共通規律（両タスク）

- `prompts/_shared/preamble.md` の不変式・禁止表現を継承。
- **禁止**：哲学的テーゼを「証明する」／西田が Gödel を「予見」／数学的新規性主張。
  両タスクとも**既知の数学のインスタンス化・検証**であって新規数学ではない。
- **「通った風を装わない」**：`lake build` の exit 0 を見ていないなら「未確認」と書く。
  実装が部分的なら正直に。失敗・未達・近似の正確な報告は成果の一部。
- 既存の検証規律（証明器と反例検証器の分離、CI の `git diff --exit-code`）を壊さない。
